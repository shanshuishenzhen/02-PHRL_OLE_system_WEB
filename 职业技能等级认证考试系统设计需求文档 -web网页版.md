局域网在线考试系统（Web 网页版）完整方案
一、项目概述
    1. 应用场景
        面向职业技能等级认证考试，部署于局域网环境，支持理论知识、操作技能等多类型考核，满足 500 人峰值（常规 100 人）同时在线考试需求。
    2. 核心目标
        保障考试过程的公平性与安全性（防作弊）
        支持稳定的高并发考试流程
        实现自动化阅卷与成绩管理
        提供完善的异常处理与数据备份机制
    3. 技术选型
        PH&RL 在线考试系统是一个基于 Python 的现代化、模块化考试管理平台，采用纯 Web 架构实现局域网内职业技能等级考试系统，考生通过浏览器访问系统完成考试全流程，无需安装独立客户端。系统基于 B/S（浏览器 / 服务器）架构，所有功能通过 HTML 网页呈现，适配 Windows 10 及以上操作系统的主流浏览器
        服务器端可以打包成Windows系统可执行文件，方便在其他服务器上快速方便部署
    3. 核心优势
        部署便捷：无需在考生终端安装软件，通过浏览器即可访问
        维护简单：服务器端统一更新，无需逐台终端升级
        跨浏览器兼容：支持主流现代浏览器，降低环境适配成本
        扩展灵活：可通过网页技术实现复杂交互与监控功能

三、核心功能模块设计
    1. 用户管理模块
        核心功能
            管理四类用户：超级管理员（系统内置、隐含）、管理员（系统配置）、考评员（考场管理）、考生（参加考试），权限分级控制
            权限分级控制（如管理员可创建考试，监考员仅查看监控）
        信息管理：
            支持批量导入考生信息：Excel/CSVExcel 格式，含姓名、身份证号、报考科目等
            维护用户状态：待审核（Pending Review）、审核通过（Approved）、审核未通过（Rejected）
        身份认证：
            基础认证：身份证号码 + 密码（密码加密存储）
            扩展认证：预留人脸识别 / 指纹识别接口（后期可开发）
        与其他模块协作：
            向所有模块提供用户身份与权限校验服务
            向考试执行模块提供考生名单与准入资格
            接收日志审计模块的用户操作记录请求
        关键需求
            支持 500 用户并发登录，响应时间≤1 秒
            考生信息加密存储，符合隐私保护要求
        会话管理（Redis 存储，超时自动登出）
            登录日志记录（IP、时间、浏览器信息）
        与其他模块协作
            为所有模块提供身份验证与权限校验
            向考试监控模块提供考生信息
            接收日志审计模块的用户操作记录请求
        界面设计
            管理员界面：用户列表（支持筛选、搜索、批量操作）
            考生界面：登录页（含验证码）、个人信息页
    2. 题库管理模块
        核心功能
        试题管理：
            支持多题型（单选、多选、判断、填空、简答、论述题等，必须以题库规则excel数据表为准）
            录入试题内容（文字、图片、附件等）
            设置标准答案与评分规则（如主观题得分点）
        分类体系：
            按科目（如 "电工"、"焊工"）、级别（初级 / 中级 / 高级）分类
            按知识点、难度（1-5 级）标签化管理
        审核流程：新试题需管理员审核通过后才可使用
        关键需求
        支持 10 万 + 试题存储，查询响应时间≤2 秒
        试题版本管理：保留修改历史，可回溯旧版本
        组卷功能：
            手动组卷：直接选择试题
            导入模板组卷：按excel导入模板生成
        试题审核：新试题需管理员审核通过后方可使用
        与其他模块协作
             向试卷管理模块提供组卷接口
            向自动阅卷模块提供标准答案
            接收日志审计模块的操作记录
        界面设计
            试题列表页（支持批量导入 / 导出）
            试题编辑页（富文本编辑器）
            组卷规则配置页
    3. 试卷管理模块
        核心功能
        试卷创建：
            设置试卷基本信息（名称、考试时长、总分、适用科目）
            关联试卷（从题库中关联试卷）
            预览试卷并调整试题顺序
        发布试卷：
            配置考试参数（是否允许提前交卷、是否显示倒计时、是否显示答案）
            配置阅卷参数（是否自动阅卷、阅卷标准）
            发布试卷，可以用于考试端、阅卷端查看    
        考试计划：
            绑定考生范围（单个 / 多个 / 考生组）
            设置考试时间（开始时间、截止时间、可考次数）
            配置考试参数（是否允许提前交卷、是否显示倒计时、是否显示答案）
        与其他模块协作
            从题库管理模块获取试题
            向考试监控模块推送考试计划
            接收日志审计模块的操作记录
        界面设计
            试卷列表页（支持复制、禁用操作）
            试卷详情页（试题预览、参数配置）
            考试计划配置页

        核心功能
        考试流程控制：
            统一发送考试开始 / 暂停 / 结束指令
            单独控制特定考生（延长时间、强制交卷）
            实时显示考生状态（未开始 / 进行中 / 已交卷 / 异常）
        实时监控：
            接收考生端定期截屏（30 秒 / 次）
            记录考生行为日志（鼠标点击、键盘输入、页面切换）
            异常行为检测与预警（切屏、长时间无操作、多开窗口）
        防作弊机制：

        与其他模块协作
            从试卷管理模块获取考试计划
            向考生端发送控制指令
            向成绩管理模块推送交卷信息
            向日志审计模块同步异常记录
        界面设计
            考试监控大屏（总览所有考生状态）
            考生详情监控页（实时截屏、行为日志）
            异常处理页（警告发送、强制操作）
    5. 考生考试模块（Web 前端）
        核心功能
        考前准备：
            身份验证与考试资格校验
            系统环境检测（浏览器兼容性、网络状态）
            考试须知确认

            试题导航（已答 / 未答 / 标记）
        安全控制：
            全屏锁定（退出全屏自动警告）
            禁用浏览器快捷键（F12、Ctrl+Shift+I 等）

            断网重连与数据恢复
        与其他模块协作
            通过 API 网关与用户管理模块完成身份验证
            从考试监控模块接收控制指令
            向成绩管理模块提交答卷
        界面设计
            考试环境检测页
            考试主界面（左侧导航 + 中间试题区 + 顶部状态栏）
            交卷确认页
    6. 成绩管理模块
        核心功能
        答卷处理：
            接收考生提交的答卷
            按题型拆分（客观题 / 主观题）
        成绩管理：
            整合自动阅卷与手动阅卷结果

        } else if (docEl.mozRequestFullScreen) {

        } else if (docEl.webkitRequestFullscreen) {

        document.addEventListener('webkitfullscreenchange', checkFullscreen);

        if (!document.fullscreenElement &&

            // 发送警告至服务器

        }
        }

    2. 实时通信实现
        采用 WebSocket 建立长连接，服务器可主动向考生端推送指令
        心跳检测机制维护连接状态，异常断开时自动重连

            负载均衡（Nginx）处理高并发请求

七、方案优势与挑战
    优势
        部署成本低：无需安装客户端，浏览器直接访问
        维护效率高：服务器端统一更新，避免终端逐个升级
        跨平台兼容：支持所有现代浏览器，不受操作系统版本限制
        开发效率高：Web 技术栈成熟，前端框架组件丰富
    挑战与应对
        挑战点	            应对措施
        浏览器安全限制	    通过前端 JS 与后端配合实现增强控制
        防作弊难度增加	    多维度监控（截屏 + 行为分析 + 环境检测）
        前端性能瓶颈	    虚拟列表、懒加载、WebWorker 分散计算压力
        网络波动影响	    完善的离线缓存与自动重连机制
八、总结
    采用 Web 网页架构实现局域网在线考试系统是可行且高效的方案，尤其适合 500 人以内规模的职业技能等级考试。通过合理的技术选型（Vue+Django+WebSocket）和严格的安全机制（实时监控、行为分析、防作弊控制），可满足考试系统的严肃性与安全性要求，同时大幅降低部署与维护成本。
    建议实施时优先完成核心功能（考试流程 + 基础防作弊），再逐步增强监控与分析能力，通过小规模试点验证后再大规模推广。
