职业技能等级认证考试系统设计需求文档
一、项目概述
    1. 应用场景
        面向职业技能等级认证考试，部署于局域网环境，支持理论知识、操作技能等多类型考核，满足 500 人峰值（常规 100 人）同时在线考试需求。
    2. 核心目标
        保障考试过程的公平性与安全性（防作弊）
        支持稳定的高并发考试流程
        实现自动化阅卷与成绩管理
        提供完善的异常处理与数据备份机制
    3. 技术选型
        PH&RL 在线考试系统是一个基于 Python 的现代化、模块化考试管理平台，采用分布式微服务架构，提供完整的局域网在线考试解决方案。
        前端：Vue.js + Element UI
        后端：Flask + SQLAlchemy
        数据库：MySQL
        缓存：Redis
        消息队列：RabbitMQ

三、模块详细设计需求
    1. 用户管理模块
        核心功能
            管理四类用户：超级管理员（系统内置、隐含）、管理员（系统配置）、考评员（考场管理）、考生（参加考试），权限分级控制
            权限分级控制（如管理员可创建考试，监考员仅查看监控）
        信息管理：
            支持批量导入考生信息：Excel/CSVExcel 格式，含姓名、身份证号、报考科目等
            维护用户状态：待审核（Pending Review）、审核通过（Approved）、审核未通过（Rejected）
        身份认证：
            基础认证：身份证号码 + 密码（密码加密存储）
            扩展认证：预留人脸识别 / 指纹识别接口（后期可开发）
        与其他模块协作：
            向所有模块提供用户身份与权限校验服务
            向考试执行模块提供考生名单与准入资格
            接收日志审计模块的用户操作记录请求
        关键需求
            支持 500 用户并发登录，响应时间≤1 秒
            考生信息加密存储，符合隐私保护要求
    2. 独立客户端模块
        核心功能：
        安全启动：
            系统环境检测（禁止运行浏览器、聊天工具、远程控制软件等）
            禁用 USB 接口、任务管理器、系统快捷键（Alt+Tab/Win 键等），调试阶段暂时不适用
            身份验证：与用户管理模块对接，验证考生身份。登录后系统内部与考试管理模块对接，验证是否有要考试的项目，有则允许登录，没有则拒绝登录并弹出提示框。
        展示试题（支持文字、图片、视频等格式）：
            提供多样化答题界面（单选 / 多选按钮、文本框、代码编辑器等）
            实时计时与剩余时间提醒
            本地缓存：本地加密缓存试题与答案，支持断点续考，考试上传数据完毕清空本地缓存
        防作弊控制：禁用快捷键、监控屏幕、限制外部访问（调试阶段留出防作弊退出窗口，以便调试）
        网络通讯：
            与服务器保持实时连接；支持断点续传，确保答案不丢失；
            自动检测网络状态，网络恢复后自动同步数据；
            与其他模块的协作：
            从 API 网关获取考试任务和试题
            向实时监控模块上传屏幕截图和行为日志
            向成绩管理模块提交最终答案
            接收考试管理模块/实时监控模块的开始 / 结束指令
        与其他模块协作
            通过 API 网关向用户管理模块提交登录请求
            从考试执行模块获取考试指令（开始 / 暂停 / 结束）
            向考试执行模块上传屏幕截图与行为日志
            向成绩管理模块提交最终答卷
        关键需求
            防篡改：客户端程序完整性校验，禁止被破解或篡改
            稳定性：连续运行 8 小时无崩溃，异常退出后可恢复答题状态
            **完全独立运行**：移除对主控台的依赖，可独立部署
            **局域网通信**：支持HTTP/HTTPS协议与服务器通信
            **图形化配置**：内置服务器配置界面
            **完整考试功能**：登录、考试列表、答题、提交
            **防作弊机制**：全屏模式、快捷键禁用
            **可打包部署**：支持PyInstaller打包为Windows可执行文件在客户机Windows系统中安装运行
    3. 试卷管理模块
        核心功能
            试卷创建：
                创建试卷（设置名称、考试时长、总分、适用科目）
                关联试题（从题库选择或自动组卷）
                支持智能组卷，根据难度分布、知识点覆盖自动生成平衡试卷
                支持试卷模板复用，快速创建相似结构的新试卷
            组卷策略：
                导入试卷：支持从Excel导入试题组成试卷，一键完成试卷创建
                规则组卷：支持从Excel组卷规则表自动组卷，按照预设规则生成
                固定组卷：手动选择试题，按题型、难度、知识点筛选生成，完全掌控试题选择
                抽题组卷：从各知识点池随机抽取指定数量试题，确保知识点覆盖均衡
            试卷预览与检查：
                提供试卷完整性检查（总分核对、重复题目检测等）
                支持试卷预览，与考生实际看到的界面一致
                提供试卷难度分析报告，帮助评估试卷质量
                支持试题顺序调整和替换功能
            考试计划：
                绑定考生范围（单个考生 / 考生组 / 全体）
                设置考试时间（开始时间、截止时间、可考次数）
                设置考试规则（允许提前交卷、是否可查看已答题目等）
                支持批量导入考生信息，高效管理大规模考试
            试卷发布：
                向考生客户端推送试卷，确保及时送达
                控制试卷查看权限，防止提前泄露
                支持定时发布和手动发布两种模式，灵活控制
                支持试卷版本控制和回滚，应对紧急情况
        与其他模块协作
            向用户管理模块提供考试计划查询接口
            向考试执行模块推送考试计划
            接收日志审计模块的操作记录
            从题库管理模块获取题目
            向成绩管理模块提供试卷评分标准
        关键需求
            性能要求：
                支持同时存储 1000+ 份试卷，组卷响应时间≤5 秒
                支持100名管理员同时操作试卷管理系统
                试卷生成与发布过程无卡顿，确保流畅用户体验
            安全性要求：
                试卷数据不可篡改（修改需留痕）
                防止试卷提前泄露，支持试卷加密存储
                权限分级控制，确保只有授权人员可访问试卷
                操作记录完整保存，支持审计追踪
            功能性要求：
                灵活性：支持多种题型组合和试卷结构定制
                可追溯：记录试卷的创建、修改、发布全过程
                高效性：支持批量操作和模板复用，提高试卷管理效率
                智能性：提供试卷难度分析和优化建议
    4. 题库管理模块
        核心功能
        试题管理：
            支持多题型（单选、多选、判断、填空、简答、论述题等，必须以题库规则excel数据表为准）
            录入试题内容（文字、图片、附件等）
            设置标准答案与评分规则（如主观题得分点）
        分类体系：
            按科目（如 "电工"、"焊工"）、级别（初级 / 中级 / 高级）分类
            按知识点、难度（1-5 级）标签化管理
        审核流程：新试题需管理员审核通过后才可使用
        与其他模块协作
            向试卷管理模块提供试题查询与组卷接口
            向自动阅卷模块提供标准答案与评分规则
            向手动阅卷模块提供主观题与评分规则
        接收日志审计模块的试题修改记录请求
        关键需求
            支持 10 万 + 试题存储，查询响应时间≤2 秒
            试题版本管理：保留修改历史，可回溯旧版本
    5. 成绩管理模块
        核心功能
            答卷处理：
                接收并存储考生提交的答卷
                按题型拆分答卷（客观题 / 主观题）
            成绩合成：
                整合自动阅卷与手动阅卷结果
                计算总分与通过率
            结果管理：
                支持成绩查询（考生查本人，管理员查全体）
                批量导出成绩表（Excel/PDF 格式）
            成绩分析（只需统计合格名单、不合格名单基础表格即可）
        与其他模块协作：
            从独立客户端模块接收答卷数据
            向自动阅卷模块分发客观题，接收客观题评分结果
            向手动阅卷模块分发主观题，接收主观题评分结果
            向用户管理模块提供考生成绩查询接口
        关键需求
            数据一致性：确保答卷与评分结果不丢失、不篡改
            实时性：客观题提交后 5 分钟内完成自动评分
    6. 阅卷中心
        自动阅卷模块（客观题）
        核心功能：
            比对考生答案与题库标准答案
            按评分规则自动计分（如单选题选对得满分，多选错选不得分）
            生成客观题得分明细
        与其他模块协作：
            从成绩管理模块获取客观题答卷
            从题库管理模块获取标准答案
            向成绩管理模块返回评分结果
        手动阅卷模块（主观题）
        核心功能：
        任务分配：按科目 / 题型自动分配给考评员
        阅卷界面：展示考生答案、标准答案、评分标准
        评分操作：支持打分、写评语、标记争议题
        复评机制：争议题可提交给其他考评员二次评分
        与其他模块协作：
            从成绩管理模块获取主观题答卷
            向成绩管理模块返回评分结果
            从用户管理模块获取考评员权限信息
        关键需求：
            防作弊：屏蔽考生信息（匿名阅卷）
            可追溯：记录每道题的阅卷人、时间、修改记录
    7. 考试执行模块
        核心功能
            考试调度：
                按试卷管理模块的计划，统一控制考试开始 / 暂停 / 结束
                向考生客户端推送考试指令（如 "距离考试结束还有 10 分钟"）
            实时监控：
                接收考生客户端的屏幕截图（每 30 秒一次）
                分析考生行为日志（鼠标点击频率、窗口切换次数等）
                异常行为预警（如频繁切屏、长时间离开）
            异常处理：
                向违规考生发送警告信息
                支持手动干预（如强制考生交卷、延长个别考生考试时间）
        与其他模块协作
            从试卷管理模块获取考试计划与考生名单
            向独立客户端模块发送考试控制指令
            向日志审计模块同步异常事件记录
            考试结束后通知成绩管理模块开始处理答卷
        关键需求
            实时性：指令从服务器到客户端延迟≤3 秒
            并发支持：同时处理 500 个客户端的监控数据
    8. API 网关模块
        核心功能
            请求路由：统一接收客户端请求，分发至对应业务模块
            安全控制：
                验证请求合法性（令牌校验、IP 白名单）
                过滤恶意请求（如 SQL 注入、频繁请求）
            流量管理：
                限制单客户端并发请求数（≤5 个）
                高峰期自动限流，保障核心功能优先响应
            日志记录：记录所有 API 调用（请求地址、参数、响应时间等）
        与其他模块协作
            为所有业务模块提供对外接口
            与用户管理模块协作进行身份认证
            向日志审计模块提供 API 调用日志
        关键需求
            高可用：支持 7×24 小时运行，故障自动切换
            性能：每秒处理≥1000 个请求，响应时间≤500ms
        启动方式
            使用统一启动脚本：`python start_unified_api_gateway.py`
            支持自定义端口：`python start_unified_api_gateway.py --port 8888`
            支持调试模式：`python start_unified_api_gateway.py --debug`
    9. 日志审计模块
        核心功能
            日志采集：
                记录所有模块的操作日志（用户登录、试卷创建、评分等）
                收集系统日志（服务器状态、网络异常、客户端崩溃等）
            查询与分析：
                按时间、模块、用户等维度检索日志
                异常操作预警（如异地登录、批量修改成绩）
            日志存储：
                日志数据保存≥5 年
                支持日志备份与归档
        与其他模块协作
            接收所有业务模块的日志数据
            向管理员提供日志查询与导出功能
        关键需求
            不可篡改：日志生成后禁止修改或删除
            完整性：确保所有关键操作均被记录

五、安全与性能要求
    1. 安全要求
    数据安全：
    传输加密：所有通讯采用 TLS 1.3 协议
    存储加密：考生答案、成绩等敏感数据加密存储
    防作弊：
    客户端防篡改：程序完整性校验，禁止调试或注入
    行为监控：多维度检测异常行为（屏幕切换、外部程序调用等）
    权限控制：严格的 RBAC 权限模型，禁止越权操作
    2. 性能要求
    并发支持：
    同时支持 500 用户登录与考试
    瞬间提交答卷峰值（500 份 / 5 分钟）无数据丢失
    响应时间：
    客户端操作响应≤1 秒
    页面加载≤3 秒
    数据查询≤2 秒
    稳定性：
    系统可用性≥99.9%
    支持连续 8 小时无间断考试
六、部署与维护要求
    1. 部署方式
    服务器：支持物理机或虚拟机部署，需冗余备份
    客户端：支持批量部署（通过域控制器或第三方工具），支持静默安装
    2. 维护要求
    数据备份：每日自动备份数据库，保留 30 天备份记录
    日志管理：定期清理过期日志（保留 1 年），支持日志归档
    故障恢复：服务器故障时 30 分钟内可切换至备用节点
七、总结
    本文档详细描述了职业技能等级认证考试系统的设计需求，包括系统架构、模块设计、业务流程、安全性能要求、部署维护要求以及技术实现建议。通过分阶段实施，确保系统能够稳定、安全、高效地支持职业技能等级认证考试的全流程管理。 
    核心优势在于：

        完善的离线与异常处理，适应复杂局域网环境
        自动化与人工协作结合的阅卷流程，兼顾效率与准确性
    

八、全局约定
    1."QUESTION ID"格式
        **格式**：`题库名称(3部分)-题型代码-一级代码-二级代码-三级代码-知识点代码-顺序号`
        **示例**：`BWGL-3-LL-B-A-B-C-001-002`
        **解析**：
        - `BWGL-3-LL`：题库名称代码（保卫管理员三级理论）
        - `B`：题型代码（单选题）
        - `A-B-C`：一级-二级-三级代码
        - `001`：知识点代码
        - `002`：顺序号
        **总共9个部分**，用连字符分隔

    2."API接口规范"
         **统一API标准**：所有模块间通信必须遵循统一API接口规范，详见《UNIFIED_API_SYSTEM_GUIDE.md》文档。
         **版本控制**：所有API接口必须包含版本信息，确保向后兼容性。
         **错误处理**：统一的错误码和错误信息格式，便于调试和问题定位。
         **认证授权**：统一的认证机制和权限控制策略。
 
     3."报考科目与题库名称"字段的内部关联
        **用户管理-用户导入模板："报考科目"字段**
        **题库管理-题库模板："题库名称"字段**
        **考试管理-组卷规则模板："题库名称"字段**
        **以上3个模块中的相应字段强相关，不得随意变更**
        **其他所有功能模块API都以此字段为联络基准**

    3.用户管理模块
        **默认内置隐含的超级用户，具有系统管理员权限，不能删除。在所有需要账号密码登录的模块都首先赋予超级用户所有用户的权限，然后再分配其他用户的权限**
        **调试阶段，超级用户名：admin，密码：123456**
        **发行阶段，超级用户名：phrladmin，密码：由系统内部设置，隐藏用户名及密码**

    4.全局基础模板
        **Temples文件夹下存储基础全局模板，包括：用户管理模板、题库管理模板、组卷规则模板、考试管理模板、成绩管理模板、日志审计模板**
        **代码设计和API控制必须以此模板为基础，不得随意变更** 
            **用户管理模板：用户导入模板、用户导出模板**
            **题库管理模板：题库导入模板、题库导出模板**
            **组卷规则模板：组卷规则导入模板、组卷规则导出模板**
            **考试管理模板（待开发）：考试计划导入模板、考试计划导出模板**
            **成绩管理模板（待开发）：成绩查询模板、成绩导出模板**
              **成绩查询/导出模板：**
                 **考生ID**：考生的唯一标识符
                 **考生姓名**：考生的姓名
                 **考试科目**：考试的科目名称
                 **考试时间**：考试的具体日期和时间
                 **总分**：考生的总分数
                 **各题得分**：考生在每道题目上的得分
                 **是否及格**：考生是否达到及格线
                 **排名**：考生在本次考试中的排名
                 **证书编号**：如果考生及格，分配的证书编号
            **日志审计模板（待开发）：日志查询模板、日志导出模板**
               **日志查询模板：**
                  **操作时间**：日志记录的时间，精确到秒
                  **操作用户**：执行操作的用户名
                  **操作类型**：如登录、修改、删除等
                  **操作对象**：被操作的对象，如用户、试题、考试等
                  **操作内容**：详细的操作描述
                  **操作结果**：成功或失败
                  **IP地址**：操作者的IP地址
                  **设备信息**：操作者的设备信息
               **日志导出模板：**
                  与日志查询模板相同，但增加导出时间和导出人员信息
        **待开发模板**
            **考试计划导入模板：**
                **考试名称**：考试的正式名称，如"2023年下半年职业技能等级认证考试"
                **考试时间**：考试的具体日期和时间，如"2023年7月1日 9:00 - 12:00"
                **考试时长**：考试的总时长，单位为分钟，如"120"（120分钟）
                **考试科目**：考试的科目，如"法律、行政、财务、工程"等
                **考试类型**：考试的类型，如"理论考试"、"实践考试"
                **考试人数**：考试的总人数，如"500"
                **及格线**：及格分数线，如"60"（60分及格）
                **考试规则**：考试的具体规则，如"考试时间内完成所有题目"

九、技术架构与实现策略

    1. Agent组合应用架构
        **核心组件**
            **Agent组合应用模块**：负责协调和管理各个功能模块之间的交互，实现业务流程的自动化和智能化。
            **统一API网关**：所有模块间的通信必须通过API网关进行，确保安全性和一致性。
            **统一API接口池**：集中管理所有API接口，便于维护和扩展。
            **后端服务模块**：包括用户管理、题库管理、试卷管理、考试执行、成绩管理等核心业务模块。

        **架构优势**
            **高内聚低耦合**：各模块通过标准化接口通信，降低系统复杂度。
            **可扩展性**：支持新功能模块的快速集成。
            **可维护性**：统一的接口标准和文档，降低维护成本。
            **高可用性**：支持模块级别的故障隔离和恢复。

        **版本控制策略**
            **URL路径版本**：如`/api/v1/questions`、`/api/v2/questions`
            **向后兼容**：新版本必须支持旧版本的请求格式
            **版本过渡期**：新旧版本并行运行，确保平滑过渡

        **错误处理机制**
            **统一错误码**：如400表示请求错误，500表示服务器错误
            **详细错误信息**：包含错误位置、原因和可能的解决方案
            **日志关联**：错误信息与系统日志关联，便于问题定位

    3. 监控与管理
        **健康检查**
            **接口健康监控**：定期检查API接口的可用性和响应时间
            **资源使用监控**：监控系统资源使用情况，如CPU、内存、磁盘等
            **告警机制**：当监控指标超过阈值时，自动触发告警

        **性能分析**
            **接口响应时间**：监控各接口的响应时间，识别性能瓶颈
            **并发处理能力**：测试系统在高并发情况下的表现
            **资源消耗分析**：分析各模块的资源消耗情况，优化资源分配

    4. 测试与验证
        **单元测试**
            **接口测试**：验证各API接口的功能正确性
            **边界测试**：测试各种边界条件下的系统行为
            **异常测试**：测试系统对异常情况的处理能力

        **集成测试**
            **模块间交互**：测试各模块之间的交互是否正常
            **端到端测试**：模拟真实用户场景，测试完整业务流程
            **性能测试**：测试系统在高负载下的性能表现

    5. 故障排除与优化
        **常见问题**
            **端口冲突**：API网关或服务端口被占用
            **服务无法启动**：配置错误或依赖缺失
            **API调用失败**：网络问题或参数错误

        **优化策略**
            **缓存机制**：对频繁访问的数据进行缓存，提高响应速度
            **负载均衡**：在多服务器环境下，合理分配请求负载
            **代码优化**：定期进行代码审查和优化，提高系统性能

    6. 安全策略
        **认证与授权**
            **JWT认证**：使用JSON Web Token进行用户认证
            **角色权限控制**：基于角色的访问控制，确保数据安全
            **API访问限制**：限制API的访问频率，防止恶意攻击

        **数据安全**
            **传输加密**：使用HTTPS加密传输数据
            **存储加密**：敏感数据加密存储
            **数据备份**：定期备份重要数据，防止数据丢失

    7. 部署与维护
        **部署流程**
            **环境准备**：准备开发、测试和生产环境
            **代码部署**：使用自动化工具进行代码部署
            **配置管理**：统一管理各环境的配置信息

        **维护策略**
            **定期更新**：定期更新系统组件和依赖库
            **性能监控**：持续监控系统性能，及时发现问题
            **问题跟踪**：建立问题跟踪机制，确保问题得到及时解决
    8. 局域网在线考试系统采用 HTML 网页架构
        前端技术栈
            框架：React + TypeScript（或 Vue 3）
            状态管理：Redux Toolkit（或 Pinia）
            UI 组件库：Ant Design/Element Plus（提供丰富表单组件）
            实时通信：Socket.IO（兼容 WebSocket 和长轮询）
            本地存储：IndexedDB（大容量存储）+ localStorage（简单配置）
        后端技术栈
            语言 / 框架：Python + Django REST Framework（或 Node.js + Express）
            实时服务：WebSocket 服务器（如 Django Channels 或 Node.js 的 ws 库）
            缓存：Redis（存储在线用户状态、考试实时数据）

        考生端核心界面

            成绩管理界面：可视化报表 + 导出功能
